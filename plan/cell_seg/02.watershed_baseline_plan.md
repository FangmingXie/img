# Plan 02 — Watershed baseline for 3D nuclei segmentation

## Context

A classical baseline for segmenting nuclei (DAPI = **C0**) in
`links/cell_seg/cell_segmentation_Saumya.tif` (ZCYX = 39×4×262×266, uint16). It
serves two purposes: (1) an early result to eyeball alongside the deep-learning
run, and (2) a reference to benchmark the Cellpose result against
(see `plan/cell_seg/01.nuclei_segmentation_plan.md`). Image facts and channel
identities are in `links/cell_seg/README.md` (C0 = DAPI, to be visually confirmed).

Uses only standard scientific-Python packages (numpy, scipy, scikit-image,
tifffile, pandas, matplotlib), all run from the project env `img` — **not**
borrowed from another conda env.

## Method — distance-transform marker-controlled watershed (3D)

Standard pipeline for round, blob-like nuclei. Steps, with the scikit-image /
scipy function for each:

1. **Load** C0 volume `(Z,Y,X)` from the TIFF (`tifffile.imread(...)[:, C0]`).
2. **Per-Z intensity normalization** — deeper confocal slices are dimmer. Rescale
   each Z-slice to its own percentile range (e.g. 1–99.8) so a single global
   threshold works across depth. (`np.percentile` per slice + `skimage.exposure.rescale_intensity`.)
3. **Smooth** — `skimage.filters.gaussian` with **anisotropic sigma**
   `(SIGMA_Z, SIGMA_XY, SIGMA_XY)` to suppress speckle without merging neighbors.
4. **Foreground mask** — threshold the smoothed volume. Default global
   `skimage.filters.threshold_otsu`; offer `threshold_local` as a fallback for
   uneven background. Result = binary `fg`.
5. **Clean mask** — `scipy.ndimage.binary_fill_holes`, then
   `skimage.morphology.remove_small_objects(min_size=MIN_VOXELS)` and an optional
   small `binary_opening` to break thin bridges.
6. **Distance transform** — `scipy.ndimage.distance_transform_edt(fg, sampling=(ANISOTROPY_Z, 1, 1))`.
   `sampling` encodes Z:XY voxel-size ratio so distances are physically sensible
   despite anisotropy (Z-step is unrecorded → `ANISOTROPY_Z` is a tunable estimate).
7. **Seeds (the over/under-segmentation knob)** — default
   `skimage.feature.peak_local_max` on the distance map with `min_distance=MIN_DISTANCE`
   and `exclude_border=False`, then label peaks (`scipy.ndimage.label`). Provide an
   alternative seed mode `skimage.morphology.h_maxima(distance, H_MAXIMA)` which is
   more robust to splitting a single nucleus into many seeds.
8. **Watershed** — `skimage.segmentation.watershed(-distance, markers, mask=fg)`
   → integer label volume.
9. **Post-filter** — drop objects below `MIN_VOXELS` / above `MAX_VOXELS`
   (debris / merged blobs); relabel sequentially (`skimage.segmentation.relabel_sequential`).

### 3D vs 2D+stitch
Default is **full 3D** watershed with `ANISOTROPY_Z`. Because the true Z-step is
unknown, also expose a `MODE = "3d" | "slice_stitch"` switch: in `slice_stitch`,
run steps 3–8 per Z-slice in 2D, then link labels across adjacent slices by IoU
overlap (`> STITCH_IOU`) — robust when anisotropy is badly misestimated. Start
with `3d`; switch if Z-merging looks wrong.

### `02b.segment_watershed.py`
Top-of-file config block (all tunables in one place):
```python
PROJECT_ROOT      = os.path.dirname(os.path.dirname(os.path.dirname(os.path.abspath(__file__))))
INPUT_FILE        = os.path.join(PROJECT_ROOT, 'links', 'cell_seg', 'cell_segmentation_Saumya.tif')
OUT_RES_DIR       = os.path.join(PROJECT_ROOT, 'local_data', 'res', 'cell_seg')
OUT_LABELS        = os.path.join(OUT_RES_DIR, '02b.nuclei_labels_watershed.tif')

NUC_CHANNEL       = 0          # C0 = DAPI
MODE              = "3d"       # "3d" | "slice_stitch"
ANISOTROPY_Z      = 3.0        # estimated Z:XY voxel ratio (Z-step unrecorded)
SIGMA_XY          = 1.5
SIGMA_Z           = 1.0
PNORM_LO, PNORM_HI= 1.0, 99.8  # per-slice percentile normalization
THRESHOLD         = "otsu"     # "otsu" | "local"
MIN_DISTANCE      = 10         # peak_local_max separation (px)
DIST_SMOOTH       = 2.0        # gaussian sigma (XY) on distance map before seeding
SEED_MODE         = "peak"     # "peak" | "hmaxima"
H_MAXIMA          = 2.0
MIN_VOXELS        = 50
MAX_VOXELS        = 50000
STITCH_IOU        = 0.3        # slice_stitch mode only
```
Steps 1–9 above; save the label volume as ImageJ-compatible TIFF
(`tifffile.imwrite(OUT_LABELS, labels.astype('uint16' if labels.max()<65535 else 'uint32'), imagej=True)`).
Print the object count.

### `03.measure_and_qc.py` (shared with Plan 01 — generic over a label TIFF)
Make it take the label path as an argument / config constant so the *same* script
quantifies both watershed and Cellpose masks:
- `skimage.measure.regionprops_table(labels, intensity_image=<all 4 channels>)` →
  per-cell CSV (`label`, centroid z/y/x, `area` voxels, `mean_intensity` per C0–C3)
  → `res/cell_seg/03.nuclei_measurements_watershed.csv`.
- Count + volume-distribution summary → `res/cell_seg/03.count_summary_watershed.txt`.
- QC overlays to `fig/cell_seg/`: `skimage.segmentation.find_boundaries` of the
  mask drawn over C0 for a few representative Z-slices and the max-Z-projection.

## Tuning order (most → least impactful)
1. `MIN_DISTANCE` / `DIST_SMOOTH` / `SEED_MODE`+`H_MAXIMA` — controls split vs merge
   of touching nuclei. Smoothing the distance map (`DIST_SMOOTH`) before seeding is
   the key fix for sub-nuclear DAPI texture that otherwise causes over-segmentation.
2. `THRESHOLD` (otsu vs local) and per-slice normalization — controls what counts as foreground.
3. `ANISOTROPY_Z` — Z-merging behavior (or switch to `slice_stitch`).
4. `SIGMA_*`, `MIN_VOXELS`/`MAX_VOXELS` — denoise + debris/merge filtering.

## Known limitations (why this is a *baseline*)
- Watershed over-segments large/irregular nuclei and under-segments tightly
  touching clusters; the seed step is sensitive to tuning.
- No learned shape prior → expect Cellpose/StarDist to beat it on dense regions.
- Unknown Z-step makes true-3D distances approximate. Good enough for sanity-check
  counts and morphology, not as a final quantitative result without spacing.
