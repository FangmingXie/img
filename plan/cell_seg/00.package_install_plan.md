# Plan 00 — Package installation for the `img` env

## Context

Prerequisite for the cell-segmentation work (Plans 01 & 02). All image-processing
packages go into the **single project env `img`** (currently bare py3.11) — no
borrowing from other conda envs. This plan is the one place package installation
is defined; the other plans just assume `img` is ready.

**Policy (from CLAUDE.md):** do not install without explicit user OK; snapshot the
current package list to `env/` before installing; try conda first, pip later.

System facts: GPU = 4× NVIDIA RTX 6000 Ada (48 GB, idle), so a CUDA build of
PyTorch is appropriate for Cellpose.

## Package groups

**Group A — scientific stack** (watershed baseline, channel preview, QC/measurement;
Plan 02 + the shared `03.measure_and_qc.py` + `01.preview_channels.py`):
`numpy  scipy  scikit-image  tifffile  pandas  matplotlib`

**Group B — Cellpose / deep learning** (Plan 01):
`pytorch` (CUDA build) + `cellpose` (Cellpose-SAM, v4).

## Steps

### 0. Snapshot current env (before any change)
```
conda list -n img > env/img_conda_list_<date>.txt
conda run -n img pip freeze > env/img_pip_freeze_<date>.txt
```

### 1. Group A via conda (conda-first; all on conda-forge)
```
conda install -n img -c conda-forge numpy scipy scikit-image tifffile pandas matplotlib
```
Verify: `conda run -n img python -c "import numpy,scipy,skimage,tifffile,pandas,matplotlib; print('A ok')"`

### 2. PyTorch (GPU) — conda first, pip fallback
Try conda first per policy:
```
conda install -n img -c pytorch -c nvidia pytorch pytorch-cuda=12.1
```
If the solver conflicts with the conda-forge base (mixing channels can be fragile),
fall back to the official pip CUDA wheel:
```
conda run -n img pip install torch --index-url https://download.pytorch.org/whl/cu121
```
Verify GPU: `conda run -n img python -c "import torch; print(torch.__version__, torch.cuda.is_available())"`
(must print `True`). Pin the CUDA series to what the driver supports (cu121/cu124).

### 3. Cellpose via pip
Cellpose (v4 / Cellpose-SAM) is distributed on PyPI and is the maintainer-recommended
path; install after torch so it doesn't pull a CPU-only torch:
```
conda run -n img pip install cellpose
```
Verify: `conda run -n img python -c "import cellpose; from cellpose import models; print('cellpose', cellpose.version)"`
(first model use downloads the `cpsam` weights — note the one-time download).

### 4. Re-snapshot after install
```
conda list -n img > env/img_conda_list_<date>_post_install.txt
conda run -n img pip freeze > env/img_pip_freeze_<date>_post_install.txt
```

## Optional / deferred (install only if Cellpose quality is insufficient)
- **StarDist 2D** alternative (Plan 01): `pip install stardist csbdeep tensorflow`
  — heavier (TensorFlow); defer until needed.
- **InstanSeg**: `pip install instanseg-torch` — defer.

## Verification (all-in-one)
```
conda run -n img python -c "import numpy,scipy,skimage,tifffile,pandas,matplotlib,torch,cellpose; \
print('imports ok; cuda=', torch.cuda.is_available())"
```
Expect all imports to succeed and `cuda= True`. Confirm `env/` holds both the
pre- and post-install snapshots.

## Notes / risks
- **Channel mixing**: conda-forge base + pytorch/nvidia channels can clash; the pip
  fallback for torch sidesteps this and is well-supported on this GPU.
- **CUDA series** must match the driver (`nvidia-smi`); adjust `cu121`→`cu124` etc.
- After this plan, *all* segmentation scripts run with `conda run -n img ...`.
