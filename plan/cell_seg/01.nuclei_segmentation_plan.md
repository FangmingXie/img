# Plan 01 ‚Äî 3D nuclei segmentation of `cell_segmentation_Saumya.tif`

## Context

`links/cell_seg/cell_segmentation_Saumya.tif` ‚Üí `/home/qlyu/mydata/data/jainlab26_img/cell_segmentation_Saumya.tif`
is a deconvolved Nikon AX confocal stack of a *Drosophila* sample
(`9B08_slp1_RNAiv20_24h_slp1_beatiia ... lobe1`). Goal: segment **nuclei** in 3D
and produce label masks, a per-cell measurement table, QC overlays, and a count
summary. GPU is plentiful (4√ó RTX 6000 Ada, 48 GB, idle), so a deep-learning
segmenter is the right call.

### Image facts (measured)
- Shape **ZCYX = (39, 4, 262, 266)**, `uint16`, ImageJ hyperstack, unit = micron.
- Originally 1024√ó1024 in the source `.nd2`; saved here downsampled to ~262√ó266 (XY).
- **Z-step not recorded** in metadata (`spacing: None`).
- Per-channel signal: **C0** dense/uniform (mean 78) ‚Üí DAPI/nuclei; C1 bright/dense
  (mean 116); C2 sparse (mean 7); C3 very sparse (51% nonzero) ‚Äî specific markers.

### Decisions (confirmed with user)
- **Target:** nuclei, on **channel C0 (DAPI)**.
- **Z handling:** Z-step unknown ‚Üí segment **2D per-slice + stitch across Z**
  (`do_3D=False, stitch_threshold>0`), which needs no anisotropy. Keep true-3D
  (`do_3D=True` with an estimated anisotropy) as a fallback toggle.
- **Deliverables:** label-mask TIFF ¬∑ per-cell CSV ¬∑ QC overlay PNGs ¬∑ count summary.

## Tool choice

**Primary: Cellpose-SAM (Cellpose v4, `cpsam` model).** Generalist, no training
needed, strong on DAPI nuclei, native 3D + 2D-stitch modes, GPU. This is the
"existing tool" the user pointed at and the best default here.

**Alternatives considered (documented, not run by default):**
- **scikit-image watershed baseline** (Gaussian ‚Üí threshold ‚Üí distance transform ‚Üí
  `peak_local_max` ‚Üí `watershed`). Uses standard scientific-Python packages
  installed into the `img` env (see `plan/cell_seg/02.watershed_baseline_plan.md`).
  Run first as a sanity check / fallback if the Cellpose install slips.
- **StarDist 2D `2D_versatile_fluo` + Z-stitch** ‚Äî excellent for round, densely
  packed fluorescent nuclei; good second opinion if Cellpose over/under-merges.
  (StarDist *3D* would need training data, so we use the 2D pretrained model per-slice.)
- **InstanSeg** (2024) ‚Äî fast nuclei/cell model with 3D support; mention only.

Recommendation: start with the **watershed baseline** for an immediate look, then
**Cellpose-SAM** as the production result; reach for **StarDist 2D+stitch** only if
Cellpose quality is unsatisfactory.

## Cellpose API (as installed ‚Äî cellpose 4.1.1, verified)

- **Model:** `from cellpose import models; model = models.CellposeModel(gpu=True)`.
  Default `pretrained_model='cpsam'` (Cellpose-SAM). The old `models.Cellpose`
  class **no longer exists** ‚Äî do not use `model_type=...`.
- **Eval:** `masks, flows, styles = model.eval(x, ...)` ‚Äî returns **3** values
  (no `diams`). Relevant kwargs and their defaults:
  `z_axis=None, channel_axis=None, normalize=True, diameter=None,
  flow_threshold=0.4, cellprob_threshold=0.0, do_3D=False, anisotropy=None,
  stitch_threshold=0.0, min_size=15, batch_size=8`.
- **Single-channel input:** pass the C0 volume as `(Z, Y, X)` with `z_axis=0` and
  `channel_axis=None`; cpsam handles grayscale internally (the old `channels=[...]`
  arg is obsolete for cpsam). `normalize=True` lets cpsam percentile-normalize.
- **Diameter:** cpsam is trained at `diam_mean‚âà30 px`; `diameter=None` uses that.
  Nuclei here span ~30‚Äì40 px in XY (from the watershed QC), so `None` should be
  close; provide an explicit `DIAMETER` knob for tuning.
- **2D+stitch (primary):** `do_3D=False, stitch_threshold=0.4, z_axis=0` ‚Äî segments
  each Z-slice in 2D, links across Z by mask IoU. No anisotropy needed (matches the
  unknown-Z decision). **3D fallback:** `do_3D=True, anisotropy=ANISOTROPY, z_axis=0`.

## Pipeline ‚Äî scripts under `scripts/cell_seg/` (numbered, paths CAPS at top)

All read `links/cell_seg/cell_segmentation_Saumya.tif`; write results to
`local_data/res/cell_seg/` and figures to `local_data/fig/cell_seg/`. Each script
defines `PROJECT_ROOT` / `INPUT_FILE` / `OUT_*` at the top (repo convention) and
fails fast on missing inputs. Run with `conda run --no-capture-output -n img python -u ...`.

**`01.preview_channels.py`** ‚Äî load stack, write per-channel max-Z-projection PNGs
(plus a mid-Z-slice grid) to `fig/cell_seg/` so channel identities (esp. confirming
C0=DAPI) are visually verified. Also print the per-channel intensity stats table.

**Two segmentation scripts** (2D and 3D kept separate, per request), both reading
C0 `(Z,Y,X)` as float32, building `model = CellposeModel(gpu=True)`, applying
per-Z percentile normalization (counters depth dimming), saving a relabelled
ImageJ-compatible TIFF, and printing elapsed time + nucleus count. Shared config
defaults: `DIAMETER=None`, `FLOW_THRESHOLD=0.4`, `CELLPROB_THRESHOLD=-1.0`
(lower ‚Üí recover dimmer/deeper nuclei), `MIN_SIZE=50`, `BATCH_SIZE=16`,
`PER_Z_NORM=True` (so `normalize=False` is passed to `eval`).

- **`02.segment_nuclei_cellpose.py`** ‚Äî 2D per-slice + Z-stitch
  (`do_3D=False, stitch_threshold=0.4, z_axis=0`). No anisotropy needed.
  ‚Üí `02.nuclei_labels_cellpose.tif`. **As-run: 50 nuclei, 5.3 s.** Cleaner contours;
  tends to lose nuclei in dim/blurry deep slices.
- **`02c.segment_nuclei_cellpose_3d.py`** ‚Äî true 3D
  (`do_3D=True, anisotropy=2.78, z_axis=0`). ‚Üí `02c.nuclei_labels_cellpose3d.tif`.
  Recovers deep nuclei but over-fragments ‚Äî and this persists with the correct
  anisotropy (see report), so it is deprioritized.
- **`02d.segment_nuclei_cellpose_enhanced.py`** ‚Äî `02` plus per-slice preprocessing
  (white-tophat background subtraction, light gaussian denoise, soft CLAHE) and
  `augment=True`, all as config toggles. Aims to recover deep/dim nuclei. Reduces XY
  merging but does not fix the deepest slices (see report ¬ß4). ‚Üí `02d.nuclei_labels_cellpose_enhanced.tif`.
- **`02e.segment_nuclei_cellpose_zcrop.py`** ‚Äî best procedure (`02` baseline) on the
  Z-cropped stack (drops the deep low-contrast tail z‚â•27); output padded back to full
  Z. 41 nuclei, median 9.36 ¬µm¬≥. ‚Üí `02e.nuclei_labels_cellpose_zcrop.tif`.
- **`02f.split_doublets.py`** ‚Äî post-hoc split of oversized masks via intensity-guided
  watershed (seed from smoothed-DAPI peaks; cuts in the valleys between nuclei). Runs
  on a label TIFF (default `02e`). **Current best pipeline (`02e`‚Üí`02f`): 52 nuclei,
  median 10.95 ¬µm¬≥, max 51 ¬µm¬≥** ‚Äî see `report/cell_seg/02.zcrop_deep_slice_results.md`.
  ‚Üí `02f.nuclei_labels_split.tif`.

**`03.measure_and_qc.py`** ‚Äî *already implemented* (Plan 02); generic over any label
TIFF via a CLI arg. Reuse as-is:
`conda run -n img python -u scripts/cell_seg/03.measure_and_qc.py <OUT_LABELS>`
‚Üí writes `03.nuclei_measurements_<tag>.csv`, `03.count_summary_<tag>.txt`, and QC
overlays, where `<tag>` is derived from the label filename (e.g. `cellpose`).
(Minor: improve the tag derivation to the last `_`-token so
`02.nuclei_labels_cellpose.tif` ‚Üí tag `cellpose`.)

**`02b.segment_watershed.py`** ‚Äî the classical baseline (Plan 02), already
implemented; gives the comparison mask (`tag=watershed`).

## Tuning knobs (expected iteration, most ‚Üí least impactful)
1. `DIAMETER` ‚Äî if cpsam under/over-merges, set to the measured XY nucleus diameter.
2. `STITCH_THRESHOLD` ‚Äî raise to split Z-merged nuclei, lower to connect fragments.
3. `CELLPROB_THRESHOLD` (lower = more/larger masks) and `FLOW_THRESHOLD`
   (higher = keep more, less shape-strict) ‚Äî recall vs precision.
4. Run `02c` (true 3D) with a corrected `ANISOTROPY` if 2D+stitch under-merges along
   Z ‚Äî but it needs the real Z:XY ratio to avoid over-fragmentation.
